### **Key Takeaways:**
1. **JavaScript** is the core language, while **Node.js** is a runtime to execute JS outside browsers.
2. **JavaScript** is for client-side, **Node.js** is for server-side.
3. They share syntax but have different APIs (e.g., no `document` in Node.js, no `fs` in browser JS).

### **Example Code Comparison:**
#### **Browser JavaScript (Frontend)**
```javascript
// Manipulate the DOM
document.getElementById("btn").addEventListener("click", () => {
  alert("Hello from the browser!");
});

// Use browser APIs
fetch("https://api.example.com/data")
  .then(response => response.json());
```

#### **Node.js (Backend)**
```javascript
// Create a server
const http = require('http');
http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type': 'text/plain'});
  res.end('Hello from Node.js!');
}).listen(3000);

// Access files
const fs = require('fs');
fs.readFile('file.txt', 'utf8', (err, data) => {
  console.log(data);
});
```

### **When to Use Which?**
- Use **JavaScript** for frontend (React, Angular, Vue).
- Use **Node.js** for backend (Express, NestJS), scripts, or tools (Webpack, ESLint).
