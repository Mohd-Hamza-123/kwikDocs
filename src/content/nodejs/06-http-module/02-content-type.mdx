 One of the important aspects of HTTP communication is specifying the **Content-Type header** in responses, which tells the client how to interpret the data being sent.


### Common Content Types in HTTP

Content types are based on the [MIME types](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types). Below are common examples:

#### 1. **Textual Data**

* `text/plain`: Plain text.
* `text/html`: HTML documents.
* `text/css`: CSS stylesheets.
* `text/javascript`: JavaScript code.

```javascript
const server = http.createServer((request, response) => {
    response.setHeader('Content-Type', 'text/plain')
    response.write("This is a response from server")
    response.end()
})
```

```javascript
const server = http.createServer((request, response) => {
    response.setHeader('Content-Type', 'text/html')
    response.write("<h1>This is a response from server</h1>")
    response.end()
})
```


#### 2. **JSON and XML**

* `application/json`: JSON data.
* `application/xml`: XML data.

```javascript
const server = http.createServer((request, response) => {
    response.setHeader('Content-Type', 'application/json')
    response.write(JSON.stringify({ name: 'John', age: 30 }))
    response.end()
})
```

#### 3. **Form Data**

* `application/x-www-form-urlencoded`: Encoded form data (key-value pairs).
* `multipart/form-data`: Used for forms with file uploads.

#### 4. **Media**

* `image/png`: PNG images.
* `image/jpeg`: JPEG images.
* `image/gif`: GIF images.
* `video/mp4`: MP4 videos.
* `audio/mpeg`: MP3 audio.

#### 5. **Binary Files**

* `application/octet-stream`: Binary data (generic).
* `application/pdf`: PDF files.
* `application/zip`: ZIP archives.

#### 6. **Other**

* `application/vnd.api+json`: For APIs that follow the JSON\:API specification.
* `text/event-stream`: For Server-Sent Events (SSE).
* `application/javascript`: JavaScript files.
