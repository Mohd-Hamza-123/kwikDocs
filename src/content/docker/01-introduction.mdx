### ðŸ’» **Real-World Example (Web App Deployment)**

Suppose you're building a **React frontend**, **Node.js backend**, and **MongoDB** database.

* Without Docker:

  * You manually install Node.js, MongoDB, configure ports, deal with OS compatibility.
  * Moving this to production or sharing with another dev is painful.

* With Docker:

  * You create `Dockerfile`s for each service and use `docker-compose.yml` to manage them.
  * One command (`docker-compose up`) spins up the entire app with all services working together.
