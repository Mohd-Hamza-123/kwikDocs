In **React Router v7** , **nested navigation** is done with **nested routes** and the `<Outlet />` component.


### âœ… Step 1: Define a Layout Component with `<Outlet />`

A layout is a parent route that wraps children. The `<Outlet />` is where child routes will be rendered.

```tsx
import { Link, Outlet } from "react-router-dom";

function DashboardLayout() {
  return (
    <div>
      <nav>
        <Link to="profile">Profile</Link>
        <Link to="settings">Settings</Link>
      </nav>

      {/* Child routes will render here */}
      <Outlet />
    </div>
  );
}
```

---

### âœ… Step 2: Set Up Nested Routes

```tsx
import { Routes, Route } from "react-router-dom";

function App() {
  return (
    <Routes>
      <Route path="/" Component={Home} />

      {/* Parent route with children */}
      <Route path="/dashboard" Component={DashboardLayout}>
        <Route path="profile" Component={Profile} />
        <Route path="settings" Component={Settings} />
      </Route>

      {/* Catch-all for 404 */}
      <Route path="*" Component={NotFound} />
    </Routes>
  );
}
```

### ðŸ”„ Index Routes (Default Child Route)

If you want a default page when visiting `/dashboard`, add an **index route**:

```tsx
<Route path="/dashboard" Component={DashboardLayout}>
  <Route index Component={Profile} />   {/* default route */}
  <Route path="settings" Component={Settings} />
</Route>
```

Now `/dashboard` â†’ automatically shows `Profile`.


**Summary**

* Use `<Outlet />` in parent layouts.
* Define child routes inside `<Route path="parent">...</Route>`.
* Use `index` for default child routes.
