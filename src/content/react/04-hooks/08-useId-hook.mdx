`useId` is a **React built-in hook** introduced in **React 18**.

Itâ€™s used to generate a **stable, unique ID** that:

* Is consistent across the server and client (important for SSR/Next.js).
* Stays the same between re-renders of the component.
* Can be used to link form inputs and labels, or generate unique element IDs when needed.


### ðŸ”¹ Syntax of useId

```jsx
const id = useId();
```

* `id` will be a unique string like: `"r:0:1"`, `"r:0:2"`, etc.
* The value is guaranteed to be stable across renders.

---

**Example 1: Label + Input Association**

```jsx
import React, { useId } from "react";

export default function NameField() {
  const id = useId();

  return (
    <div>
      <label htmlFor={id}>Name:</label>
      <input id={id} type="text" />
    </div>
  );
}
```

âœ” The `label` and `input` are linked by the same unique ID.
âœ” No risk of ID collisions even if you render many fields.

---

**Example 2: Multiple Instances of a Component**

```jsx
import React, { useId } from "react";

function Checkbox({ label }) {
  const id = useId();
  return (
    <div>
      <input type="checkbox" id={id} />
      <label htmlFor={id}>{label}</label>
    </div>
  );
}

export default function App() {
  return (
    <>
      <Checkbox label="I agree to the terms" />
      <Checkbox label="Subscribe to newsletter" />
    </>
  );
}
```

âœ” Each `Checkbox` gets its own unique `id`, so labels always point to the correct input.

```jsx
import React, { useId } from "react";

export default function DescriptionExample() {
  const id = useId();

  return (
    <div>
      <button aria-describedby={id}>Show Info</button>
      <p id={id}>Click the button to reveal more details.</p>
    </div>
  );
}
```

âœ” `aria-describedby` connects the button with the description text.

---

**Key Points**

* `useId` is **not for database IDs or keys in lists** (for that, use real IDs).
* Best for **accessibility attributes** (`id`, `htmlFor`, `aria-*`).
* Safe for **SSR + hydration** (avoids mismatches).
* Returns a **stable unique string** every render.
