Props (short for properties) are a way to pass data from one component to another in React.

```jsx
// App.jsx
import Card from './components/Card'

const App = () => {
  const productName = "iPhone 14";
  const productPrice = 1000;
  const productImage = "https://example.com/iphone-14.jpg";
  // Passing props to the Card component
  return (
    <div>
      <Card 
        name={productName}
        price={productPrice}
        image={productImage}
      />
    </div>
  )
}

export default App
```

Now Accessing props in the Card component
```jsx
// Card.jsx
import React from 'react'

const Card = (props) => {
  return (
    <div>
      <h2>{props.name}</h2>
      <p>{props.price}</p>
      <img src={props.image} alt={props.name} />
    </div>
  )
}

export default Card
```

### ðŸ”‘ Important Points about Props

* They are used to **pass data from a parent component to a child component**.
* Props are **read-only** â†’ a child component **cannot change** the props it receives.
* Props make components **reusable** by changing only the data, not the structure.
* Props are passed just like **HTML attributes** (e.g., `<Card name="iPhone" />`).
* Inside the component, props are accessed using `props.name` (or by destructuring).
* Props can be of **any type**: string, number, boolean, object, array, function, even another component.
* If a prop is not passed, it will be **undefined** (unless you set default values).
* Props help make components **dynamic** and **flexible**.
