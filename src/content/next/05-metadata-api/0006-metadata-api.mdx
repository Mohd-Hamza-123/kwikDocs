Next.js has a Metadata API that can be used to define your application metadata (e.g. meta and link tags inside your HTML head element) for improved SEO and web shareability.

```tsx
// layout.tsx | page.tsx
import type { Metadata } from 'next'
 
export const metadata: Metadata = {
  title: 'Page Title',
  description: 'Page Description',
}
 
export default function Page() {
    
}
```

## _Dynamic Metadata_

```tsx
// layout.tsx
export const metadata: Metadata = {
  title: {
    template : "%s | Title",
    default : "Default Title"
  },
  description: "Generated by create next app",
};
```
If you provide a specific page title (e.g., "Home")
It will use the template 

```nginx 
Home | Title
```

## _generateMetadata function_

Dynamic metadata depends on dynamic information, such as the current route parameters, external data, or metadata in parent segments, can be set by exporting a generateMetadata function that returns a Metadata object

```typescript
// layout.tsx
export const metadata: Metadata = {
  title: {
    template : "%s | Title",
    default : "Default Title"
  },
  description: "Generated by create next app",
};
```

```typescript
// page.tsx
export async function generateMetadata({ params }: any) {
    const {id} = await params 
    console.log(id) // output : 555
    return {
        title: `${id}`  // outputTitle : 555 | Title
    }
}

export default function page() {
    return {}
}
```

### Absolute

title.absolute can be used to provide a title that ignores title.template set in parent segments. 

```typescript
// app/layout.tsx
import type { Metadata } from 'next'
 
export const metadata: Metadata = {
  title: {
    template: '%s | Acme',
  },
}
```

```typescript
// app/about/page.tsx
import type { Metadata } from 'next'
 
export const metadata: Metadata = {
  title: {
    absolute: 'About',
  },
}
 
// Output: <title>About</title>
```
