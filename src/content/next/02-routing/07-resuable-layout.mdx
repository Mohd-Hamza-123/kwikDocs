Layouts in Next.js allow you to structure your application by creating reusable components for common elements like headers, footers, and sidebars. This ensures consistency across pages and simplifies development.

---

## Key Features of Layouts

- **Hierarchical Structure**: Layouts can be nested for parent-child routes.
- **Static and Dynamic Content**: Layouts can display both static elements (e.g., a header) and dynamic elements (e.g., breadcrumbs).
- **Improved Performance**: Shared layouts persist across page navigations, avoiding unnecessary re-renders.

---

## Creating a Basic Layout

A basic layout wraps your pages with common elements:

```tsx
// app/layout.tsx
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <header>
          <h1>My Website</h1>
        </header>
        <main>{children}</main>
        <footer>
          <p>Â© 2025 My Website</p>
        </footer>
      </body>
    </html>
  );
}
// This layout is applied to all pages in the app.
```

## Nested Layouts

For specific sections of your site, you can define nested layouts. For example, a dashboard layout:

```tsx 
// app/dashboard/layout.tsx
export default function DashboardLayout({ children }: { children: React.ReactNode }) {
  return (
    <div>
      <aside>Dashboard Sidebar</aside>
      <main>{children}</main>
    </div>
  );
}
// This layout applies to all routes under /dashboard
```
