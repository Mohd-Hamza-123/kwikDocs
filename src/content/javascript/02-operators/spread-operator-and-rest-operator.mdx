### **Spread Operator (`...`) vs Rest Operator (`...`) in JavaScript**

The **spread operator** and **rest operator** use the same syntax (`...`), but they serve opposite purposes depending on their context.

---

### **1. Spread Operator**

The **spread operator** is used to **spread** or expand elements of an array, object, or iterable into individual elements.

#### **Use Cases of the Spread Operator**

##### **a. Copying Arrays**

```javascript
const numbers = [1, 2, 3];
const newNumbers = [...numbers];
console.log(newNumbers); // Output: [1, 2, 3]
```

##### **b. Merging Arrays**

```javascript
const arr1 = [1, 2];
const arr2 = [3, 4];
const merged = [...arr1, ...arr2];
console.log(merged); // Output: [1, 2, 3, 4]
```

##### **c. Expanding Elements in Function Calls**

```javascript
const numbers = [10, 20, 30];
console.log(Math.max(...numbers)); // Output: 30
```

##### **d. Copying/Merging Objects**

```javascript
const user = { name: "Hamza", age: 25 };
const updatedUser = { ...user, location: "USA" };
console.log(updatedUser); // Output: { name: "Hamza", age: 25, location: "USA" }
```

---

### **2. Rest Operator**

The **rest operator** is used to **collect** multiple elements into a single array or object. It is often used in function arguments or destructuring.

#### **Use Cases of the Rest Operator**

##### **a. Function Arguments**

The rest operator collects all remaining arguments into a single array.

```javascript
function sum(...numbers) {
  return numbers.reduce((total, num) => total + num, 0);
}
console.log(sum(1, 2, 3, 4)); // Output: 10
```

##### **b. Destructuring Arrays**

The rest operator collects the remaining elements into a new array.

```javascript
const [first, second, ...rest] = [1, 2, 3, 4, 5];
console.log(first); // Output: 1
console.log(second); // Output: 2
console.log(rest); // Output: [3, 4, 5]
```

##### **c. Destructuring Objects**

The rest operator collects the remaining properties into a new object.

```javascript
const user = { name: "Hamza", age: 25, location: "USA" };
const { name, ...rest } = user;
console.log(name); // Output: Hamza
console.log(rest); // Output: { age: 25, location: "USA" }
```

---

### **3. Common Use Cases**

#### **a. Combining Rest and Spread Operators**

```javascript
function updateUser(user, updates) {
  return { ...user, ...updates };
}
const user = { name: "Hamza", age: 25 };
const updatedUser = updateUser(user, { age: 26, location: "USA" });
console.log(updatedUser);
// Output: { name: "Hamza", age: 26, location: "USA" }
```

#### **b. Shallow Copy Pitfall**

When copying objects, only the first level is copied. Nested objects still share references.

```javascript
const user = { name: "Hamza", address: { city: "New York" } };
const copiedUser = { ...user };
copiedUser.address.city = "Los Angeles";
console.log(user.address.city); // Output: "Los Angeles"
```

#### **c. Using Rest in Class Methods**

```javascript
class Calculator {
  add(...numbers) {
    return numbers.reduce((sum, num) => sum + num, 0);
  }
}
const calc = new Calculator();
console.log(calc.add(1, 2, 3)); // Output: 6
```

---

### **4. Key Differences Between Spread and Rest**

| **Aspect**       | **Spread Operator**                              | **Rest Operator**                                |
|------------------|--------------------------------------------------|-------------------------------------------------|
| **Purpose**      | Expands elements into individual parts.          | Collects multiple elements into an array/object.|
| **Context**      | Used in function calls, array/object literals.   | Used in function parameters and destructuring.  |
| **Example**      | `[...arr]` (spread elements of `arr`).           | `(...args)` (collects arguments).               |
