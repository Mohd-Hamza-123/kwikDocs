`localStorage` is part of the **Web Storage API** that allows you to store key-value pairs **in the browser**, with **no expiration date**. The data persists even after the browser is closed and reopened.

---

### ‚úÖ Features of localStorage:

* Stores data as **string key-value pairs**
* Data is saved **per domain** (shared across tabs on the same origin)
* **Persistent** (does not expire unless manually cleared)
* Has a limit of about **5‚Äì10MB**

---

### üìö Syntax and Methods

| Method                             | Description                              |
| ---------------------------------- | ---------------------------------------- |
| `localStorage.setItem(key, value)` | Stores a value for a given key           |
| `localStorage.getItem(key)`        | Retrieves the value for a given key      |
| `localStorage.removeItem(key)`     | Removes the key-value pair               |
| `localStorage.clear()`             | Removes **all** data in localStorage     |
| `localStorage.key(index)`          | Returns the key name at a specific index |
| `localStorage.length`              | Returns the total number of stored items |

---

### ‚úèÔ∏è Example 1: Basic Set and Get

```js
// Store data
localStorage.setItem("username", "Hamza");

// Retrieve data
const user = localStorage.getItem("username");
console.log(user); // Output: "Hamza"
```

---

### ‚úèÔ∏è Example 2: Remove and Clear

```js
localStorage.removeItem("username"); // Removes one item

localStorage.clear(); // Clears everything stored in localStorage
```

---

### ‚úèÔ∏è Example 3: Store Objects (via JSON)

Since `localStorage` only stores strings, you must convert objects to strings using `JSON.stringify()` and retrieve them using `JSON.parse()`.

```js
const user = { name: "Hamza", age: 23 };

// Store object
localStorage.setItem("user", JSON.stringify(user));

// Retrieve object
const data = JSON.parse(localStorage.getItem("user"));
console.log(data.name); // Output: "Hamza"
```

---

### ‚ö†Ô∏è Notes and Best Practices

* Always **convert objects/arrays to strings** before storing.
* Handle missing keys safely:

  ```js
  const value = localStorage.getItem("unknownKey") || "default";
  ```
* Avoid storing **sensitive data** (like passwords or tokens) in `localStorage`.
* Use `localStorage` only for **small amounts of data** (non-critical).

---

2. Go to **Application ‚Üí Storage ‚Üí localStorage**
3. View and modify key-value pairs directly
