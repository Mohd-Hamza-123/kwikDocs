Branching in Git is a powerful feature that allows you to create separate lines of development within a repository. It acts as a parallel version of your project, enabling you to work on changes without affecting the main codebase (usually the `main` or `master` branch).

---

### **Why Use Branching?**
1. **Isolate Development**:  
   Work on new features, bug fixes, or experiments without disturbing the main project.

2. **Collaboration**:  
   Developers can work on different branches simultaneously and independently, merging their work into the main branch when ready.

3. **Version Control**:  
   Represent different versions of the project, such as a release branch for deployment or a hotfix branch for urgent fixes.

4. **Testing**:  
   Test changes on a separate branch before integrating them into the main codebase.

5. **Organization**:  
   Group related changes together for better organization and clarity.

---

### **Key Branching Commands**

#### **1. Get Current Branch**
Use the following command to list all branches and highlight the one you are currently on:
```bash
git branch
```

#### **2. Create a New Branch**
To create a new branch locally:
```bash
git branch <branch-name>
# Example
git branch feature/new-feature
```

#### **3. Switch to a Branch**
Switch to an existing branch using:
```bash
git checkout <branch-name>
# Example
git checkout feature/new-feature
```

#### **4. Create and Switch to a Branch in One Step**
Combine creating and switching into a single command:
```bash
git checkout -b <branch-name>
# Example
git checkout -b feature/new-feature
```

> **Tip**: Use `git log --oneline` to verify the commit history and confirm you’re on the correct branch.

---

### **Push the Branch to the Remote Repository**

1. Push a branch to the remote repository for the first time:
   ```bash
   git push
   ```
   If your branch has no upstream tracking set, you’ll see an error like this:
   ```plaintext
   fatal: The current branch feature/new-feature has no upstream branch.
   To push the current branch and set the remote as upstream, use:
   
       git push --set-upstream origin feature/new-feature
   ```

2. Set the upstream branch and push:
   ```bash
   git push --set-upstream origin <branch-name>
   # Example
   git push --set-upstream origin feature/new-feature
   ```

---

### **Merging a Branch into `main` or `master`**

1. Switch to the main branch:
   ```bash
   git checkout main
   ```

2. Merge the branch into `main`:
   ```bash
   git merge <branch-name>
   # Example
   git merge feature/new-feature
   ```

3. Push the updated `main` branch to the remote repository:
   ```bash
   git push origin main
   ```

---

### **Renaming a Branch**
To rename a branch, use the following command while on the branch you want to rename:
```bash
git branch -m <old-branch-name> <new-branch-name>
```

### **Deleting a Branch**
To delete a branch, use the following command while on the branch you want to delete:
```bash
git branch -d <branch-name>
```

### Rebase 

Rebasing is a powerful feature in Git that allows you to integrate changes from one branch into another. It’s often used to keep a clean project history by applying commits from one branch onto another.

**Ensure you are on the branch you want to rebase**

```bash
git checkout <sub-branch-name>
git rebase main
```

### Merge vs Rebase

- **Merge**: Combines the commits from two branches and creates a new merge commit in the target branch.

- **Rebase**: Moves (or "rebases") commits from one branch on top of another branch. Rewrites commit history as if the branch started from the tip of the target branch.
